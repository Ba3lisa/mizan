# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| main branch (latest) | Yes |
| All other branches | No |

## Reporting a Vulnerability

If you discover a security vulnerability in Mizan, please report it responsibly:

1. **Do NOT open a public GitHub issue.** Security vulnerabilities should not be disclosed publicly until a fix is available.

2. **Report via GitHub Security Advisory**: Use the [private vulnerability reporting](https://github.com/Ba3lisa/mizan/security/advisories/new) feature on this repository.

3. **Alternative**: Email **essamgouda97** via GitHub's private messaging.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix for critical issues**: Within 7 days
- **Fix for non-critical issues**: Within 30 days

### Scope

The following are in scope:
- The Mizan web application (mizanmasr.com)
- The Convex backend functions
- The AI agent pipeline
- GitHub Actions workflows
- Infrastructure configuration (Terraform)

The following are out of scope:
- Third-party services (Convex, DigitalOcean, Cloudflare)
- Social engineering attacks
- Denial of service attacks

## Data Integrity

As a civic transparency platform, data integrity is a security concern. If you discover that data on Mizan has been tampered with or is being sourced from compromised endpoints, please report it using the same process above.
